Html forms

An HTML form is used to collect user input. The user input is most often sent to a server for processing.
=>All the content of the form is enclosed in the <form> tag.
=> there are many attrubutes in the form tag, some of them are :
=> action - it specifies where to send the form-data when a form is submitted.
  eg. action="https://www.w3schools.com/action_page.php"
=> method - it specifies how to send the form-data (the form-data can be sent as URL variables or as HTTP post transaction).
  method="get" sends the data as URL variables, means the names and values can be seen in the URL, so it cannot be used to send sensitive data and also its word capacity is limited.
  and method="post" sends the data as HTTP post transaction , hence it is safe.
=> name - specifies the name of the form.
=> novalidate - specifies that the form should not be validated when submitted.
=> autocomplete - specifies that the form should be autocompleted or not when submitted.
=> target - specifies where to display the response that is received after submitting the form.
=> enctype - specifies how the form-data should be encoded when submitting it to the server (only for method="post").
=> accept-charset - specifies the character encodings that are to be used for the form submission.
=> onsubmit - specifies a JavaScript to be run when the form is submitted.
=> onreset - specifies a JavaScript to be run when the form is reset.
=> accept - specifies the types of files that the server accepts (only for method="post" and enctype="multipart/form-data").

=>The <select> tag defines a drop-down list.
=>The <option> tag defines an option in a drop-down list.
>The <option> elements defines an option that can be selected.
>By default, the first item in the drop-down list is selected.
>To define a pre-selected option, add the "selected" attribute to the option tag
>to allow the user to select multiple options, add the "multiple" attribute to the <select> tag. press control key to select multiple options.
Example:
<label for="cars">Choose a car:
<select id="cars" name="cars" multiple>     =>   
<option value="volvo">Volvo</option>
<option value="saab">Saab</option>
<option value="fiat" selected>Fiat</option>
<option value="audi">Audi</option>
</select>

datalists

back to top

The <datalist> element specifies a list of pre-defined options for an <input> element.
Users will see a drop-down list of the pre-defined options as they input data and they can insert their own values along with that.
The list attribute of the <input> element, must refer to the id attribute of the <datalist> element.
Example:
<form action="/action_page.php">
<input list="browsers">
<datalist id="browsers">
<option value="Internet Explorer">   =>  
<option value="Firefox">
<option value="Chrome">
<option value="Opera">
<option value="Safari">
</datalist>
</form>

The <input> tag

back to top

=>The <input> tag is used to create interactive controls for web-based forms in order to accept data from the user, and is written inside form tag.
=> the <input> tag has many attributes, some of them are :
=> type - specifies the type of <input> element to display. eg. text,radio,checkbox, submit,reset,button,hidden,search,email,password,etc.
=> name - specifies the name of an <input> element.
=> value - specifies the value of an <input> element.
  every input element sends the name and value to the server, value is either selected or entered by the user.
  eg. <input type="text" name="username">
> In the above, name is "username" and value is the text entered by the user.

  eg. <input type="password" name="password">   =>   password:
=> reset - The <input type="reset"> defines a reset button (resets all form values to default).
  eg. <input type="reset" value="Reset">   =>  
=> button - The <input type="button"> defines a clickable button, this is not used for buttons , usually "button" tag is used for creating buttons.
  eg. <input type="button" value="Click Me">   =>  
=> hidden - The <input type="hidden"> defines a hidden input field.
A hidden field lets web developers include data that cannot be seen or modified by users when a form is submitted.
A hidden field often stores what database record that needs to be updated when the form is submitted. Note: While the value is not displayed to the user in the page's content, it is visible (and can be edited) using any browser's developer tools or "View Source" functionality. Do not use hidden inputs as a form of security!
  eg. <input type="hidden" name="hidden">   =>  
=> color - The <input type="color"> defines a color picker.
  eg. <input type="color" name="color">   =>   Pick a color:
=> date - The <input type="date"> defines a date control (year, month and day (no time)).
we can also use the min and max attributes to add restrictions to dates.
  eg. <input type="date" name="date" min="1947-08-15" max="2022-08-15">   =>   Pick a date:
=> datetime-local - The <input type="datetime-local"> defines a date and time control (year, month, day, hour, minute, and second (no time zone)).
  eg. <input type="datetime-local" name="datetime-local">   =>   Pick a date and time:
=> email - The <input type="email"> defines a field for an e-mail address.
  eg. <input type="email" name="email" placeholder="abc@xyz.com">   =>   email:
=> file - The <input type="file"> defines a file-select field and a "Browse" button (for file uploads).
  eg. <input type="file" name="file">   =>  
=> month - The <input type="month"> defines a month and year control (no time zone).
  eg. <input type="month" name="month">   =>   Pick a month:
=> number - The <input type="number"> defines a field for entering a number.
  eg. <input type="number" name="number">   =>   number:
=> range - The <input type="range"> defines a control for entering a number whose exact value is not important (like a slider control).
  eg. <input type="range" name="range">   =>  
=> search - The <input type="search"> defines a text field for entering a search string.
  eg. <input type="search" name="search">   =>   search:
=> tel - The <input type="tel"> defines a field for entering a telephone number.
  eg. <input type="tel" name="tel">   =>   tel:

=> time - The <input type="time"> defines a control for entering a time (no time zone).

input tag attributes

back to top

=> placeholder - specifies a short hint that describes the expected value of an <input> element.
=> required - specifies that an <input> element must be filled out before submitting the form.
=> disabled - specifies that an <input> element should be disabled.
=> readonly - specifies that an <input> element is read-only.
=> size - specifies the visible width of an <input> element.
The default value for size is 20.
Note: The size attribute works with the following input types: text, search, tel, url, email, and password. => maxlength - specifies the maximum number of characters allowed in an <input> element.
=> minlength - specifies the minimum number of characters allowed in an <input> element.
=> min - specifies the minimum value for an <input> element (only for type = number ,range ,date ,week ,time, datetime-local, month).
=> max - specifies the maximum value for an <input> element (only for type=number ,range ,date ,week ,time, datetime-local, month).
=> step - specifies the legal number intervals for an <input> element (only for type=number ,range ,date ,week ,time, datetime-local, month).
Example: if step="3", legal numbers could be -3, 0, 3, 6, etc.
=> pattern - specifies a regular expression that an <input> element's value should be.
The pattern attribute works with the following input types: text, date, search, url, tel, email, and password. Tip: Use the global title attribute to describe the pattern to help the user.


=> multiple - specifies that a user can enter more than one value.
=> autofocus - specifies that an <input> element should automatically get focus when the page loads.
=> autocomplete - specifies that the value of the <input> element should be automatically completed by the browser.
=> accept - specifies the types of files that the server accepts (only for type="file").
=> alt - specifies an alternate text for images (only for type="image").
=> src - specifies the URL of the image to use as a submit button (only for type="image").
=> height - specifies the height of the image in pixels (only for type="image").
=> width - specifies the width of the image in pixels (only for type="image").
=> list - specifies a list of pre-defined options for an <input> element (only for type="text", type="search", type="url", type="tel", type="email", or type="password").
=> multiple - specifies that a user can enter more than one value (only for type="email" or type="file").
=> checked - specifies that an <input> element should be pre-selected when the page loads (for type="checkbox" or type="radio").
=> selected - specifies that an <option> element should be pre-selected when the page loads (for type="checkbox" or type="radio").
=> onfocus - specifies a JavaScript to be run when an <input> element gets focus.
=> onblur - specifies a JavaScript to be run when an <input> element loses focus.
=> oninput - specifies a JavaScript to be run when an <input> element gets user input.
=> onchange - specifies a JavaScript to be run when the value of an <input> element is changed.
=> onselect - specifies a JavaScript to be run when some text is selected in an <input> element.
=> onsearch - specifies a JavaScript to be run when the user writes something in a search field (for type="search").
=> oninvalid - specifies a JavaScript to be run when an <input> element is invalid.
=> onreset - specifies a JavaScript to be run when a reset button in a form is clicked.
=> onsubmit - specifies a JavaScript to be run when a form is submitted.

=> formaction

The input formaction attribute specifies the URL of the file that will process the input when the form is submitted. Note: This attribute overrides the action attribute of the <form> element.
The formaction attribute works with the following input types: submit and image.

=> formenctype

The input formenctype attribute specifies how the form-data should be encoded when submitted (only for forms with method="post").
Note: This attribute overrides the enctype attribute of the <form> element.
The formenctype attribute works with the following input types: submit and image.

=> formmethod

The input formmethod attribute specifies the HTTP method (get or post) to use when submitting the form (only for forms with action="").
Note: This attribute overrides the method attribute of the <form> element.
The formmethod attribute works with the following input types: submit and image.
