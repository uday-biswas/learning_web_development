html blocks vs inline

=>Every HTML element has a default display value, depending on what type of element it is.
=>There are two display values: block and inline.

=>A block-level element always starts on a new line, and the browsers automatically add some space (a margin) before and after the element.
=>A block-level element always takes up the full width available (stretches out to the left and right as far as it can).
=>Two commonly used block elements are: <p> and <div>.
the other block elements are: <address>, <article>, <aside>, <blockquote>, <canvas>, <dd>, <div>, <dl>, <dt>, <fieldset>, <figcaption>, <figure>, <footer>, <form>, <h1> to <h6>, <header>, <hr>, <li>, <main>, <nav>, <noscript>, <ol>, <output>, <p>, <pre>, <section>, <table>, <tfoot>, <ul>, <video>.

=>An inline element does not start on a new line, and it only takes up as much width as necessary.
=>Two commonly used inline elements are: <a> and <span>.
the other inline elements are: <a>, <abbr>, <acronym>, <b>, <bdo>, <big>, <br>, <button>, <cite>, <code>, <dfn>, <em>, <i>, <img>, <input>, <kbd>, <label>, <map>, <object>, <q>, <s>, <samp>, <select>, <small>, <span>, <strong>, <sub>, <sup>, <textarea>, <time>, <tt>, <var>.

=> the contents inside <pre> and </pre> tags are shown as it is, without any formatting. usually it reserves the space and the line breaks.(it is not advised to use pre tag as it disturbs the responsiveness of the website.)
the user agent css style is: pre {
display: block;
font-family: monospace;
white-space: pre;
margin: 1em 0px;
}

Html class and id

=>Class
The class attribute is often used to point to a class name in a style sheet. It can also be used by a JavaScript to access and manipulate elements with the specific class name. The class attribute can be used on any HTML element, and each class can be used on multiple elements.
eg. <h2 class="city">Paris.</h2>
=>Id
The id attribute specifies a unique id for an HTML element and we cannot have more than one element with the same id in an HTML document.
The id attribute is most used to point to a specific style in a style sheet, and by JavaScript (via the HTML DOM) to manipulate the specific element with the specific id.
eg. <h2 id="city">Paris.</h2>

html image maps

=> An image map is used to make an image with clickable areas.
but the image size will not adjust with the screen size, so it is not responsive, so image with width less than 300px should be used.
The HTML <map> tag is used to define an image map, and then it is linked with the <img> tag using "usemap" attribute.
The <area> tag defines an area inside an image map, it is present inside <map> tag, and it has several attributes such as : href, shape, coords, alt, target, download.
=> shape can be circle, rect, poly, default.
>for circle, the coordinates are x,y,radius.(x,y are coordinates of center of the circle)
>for rect(rectangle), the coordinates are x1,y1,x2,y2.(x1,y1 are coordinates of top-left corner of the rectangle and x2,y2 are coordinates of bottom-right corner of the rectangle)
>for poly(polygon), the coordinates are x1,y1,x2,y2,x3,y3,....,xn,yn.(x1,y1 are coordinates of first point of the polygon and so on)
>for default, the coordinates are not required, it selects the entire region.
Example :
<img src="../images/workplace.jpg" style="float:left;margin-right:5px;" usemap="#image-map">
<map name="image-map">
<area target="_blank" alt="tea-cup" href="https://www.cup.com/" coords="337,302,46" shape="circle">
<area target="_blank" alt="laptop" title="hp laptops" href="https://www.hp.com/" coords="31,40,274,348" shape="rect">
<area target="_blank" alt="mobile" href="https://www.samsung.com/" coords= "289,172, 327,172, 334,245, 293,244" shape="poly">
<area target="_blank" alt="checking default" href="https://www.hello.com/" shape="default">

Html entities or symbols

An HTML entity is a piece of text that begins with an ampersand (&) and ends with a semicolon(;) and between is the hex code or entity name. These entities are used to display the reserved characters.
Eg., these are some html entities with how they will look on browser:

Html url(uniform resource locator)

=>a url is a unique address of a resource on the internet.
=>a url is made up of 3 parts - protocol, domain name and path.
=>the protocol is the type of communication between the browser and the server.
=>the domain name is the name of the server.
=>the path is the location of the resource on the server.
=>the url is case sensitive.
=>the url is used to identify the location of a resource on the internet.
=>A normal url looks like this - scheme:// prefix.domain:port/ path/filename
Explanation:

scheme - defines the type of Internet service (most common is http or https)
prefix - defines a domain prefix (default for http is www)
domain - defines the Internet domain name (like w3schools.com)
port - defines the port number at the host (default for http is 80)
path - defines a path at the server (If omitted: the root directory of the site)
filename - defines the name of a document or resource
